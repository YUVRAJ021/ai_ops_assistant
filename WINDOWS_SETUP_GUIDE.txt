================================================================================
       AI OPERATIONS ASSISTANT - WINDOWS 11 SETUP & DEMO GUIDE
================================================================================

This guide provides step-by-step instructions to set up and run the AI 
Operations Assistant on Windows 11.

================================================================================
                         PREREQUISITES
================================================================================

Before starting, ensure you have:

1. Python 3.10 or higher installed
   - Download from: https://www.python.org/downloads/
   - IMPORTANT: During installation, check "Add Python to PATH"

2. Git installed (optional, for cloning)
   - Download from: https://git-scm.com/download/win

3. A web browser (Chrome, Firefox, or Edge)

4. Internet connection (for API calls)

================================================================================
                    STEP 1: VERIFY PYTHON INSTALLATION
================================================================================

Open Command Prompt (cmd) or PowerShell and run:

    python --version

You should see something like: Python 3.11.x or higher

If you see an error, Python is not installed or not in PATH.

================================================================================
                    STEP 2: NAVIGATE TO PROJECT FOLDER
================================================================================

Option A - If you have the project folder:

    cd C:\path\to\ai_ops_assistant

Option B - If copying from another location:

    1. Copy the entire "ai_ops_assistant" folder to your desired location
    2. Open Command Prompt
    3. Navigate to the folder:
       
       cd C:\Users\YourUsername\Documents\ai_ops_assistant

================================================================================
                    STEP 3: CREATE VIRTUAL ENVIRONMENT
================================================================================

Run these commands in order:

    python -m venv venv

This creates a virtual environment named "venv" in the project folder.

================================================================================
                    STEP 4: ACTIVATE VIRTUAL ENVIRONMENT
================================================================================

For Command Prompt (cmd):

    venv\Scripts\activate

For PowerShell:

    .\venv\Scripts\Activate.ps1

NOTE: If you get an error in PowerShell about execution policy, run:

    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Then try activating again.

After activation, you should see (venv) at the beginning of your prompt:

    (venv) C:\Users\YourUsername\Documents\ai_ops_assistant>

================================================================================
                    STEP 5: INSTALL DEPENDENCIES
================================================================================

With the virtual environment activated, run:

    pip install -r requirements.txt

Wait for all packages to download and install. This may take 2-3 minutes.

================================================================================
                    STEP 6: CONFIGURE ENVIRONMENT VARIABLES
================================================================================

The Gemini API key is already configured in the .env file.

To verify or change it:

    1. Open .env file in Notepad:
       
       notepad .env

    2. Verify it contains:
       
       GEMINI_API_KEY=AIzaSyDrEHdAPlK3LqG4lrsYu4hG_GjJUUEoNoo

    3. Save and close

================================================================================
                    STEP 7: RUN THE APPLICATION
================================================================================

You have THREE options to run the application:

--------------------------------------------------------------------------------
OPTION A: WEB UI MODE (Recommended for Demo)
--------------------------------------------------------------------------------

Run this command:

    python main.py --api --port 8000

You should see:
    
    üöÄ Starting AI Operations Assistant API on port 8000...
    INFO:     Uvicorn running on http://0.0.0.0:8000

Now open your web browser and go to:

    http://localhost:8000

You will see a beautiful web interface!

To stop the server: Press Ctrl+C in the command prompt

--------------------------------------------------------------------------------
OPTION B: INTERACTIVE CLI MODE
--------------------------------------------------------------------------------

Run this command:

    python main.py

You will see:

    ============================================================
    ü§ñ AI Operations Assistant - Interactive Mode
    ============================================================

Type your tasks and press Enter. Type "quit" to exit.

--------------------------------------------------------------------------------
OPTION C: SINGLE COMMAND MODE
--------------------------------------------------------------------------------

Run a single task with verbose output:

    python main.py -v "What is the weather in New York?"

================================================================================
                       TROUBLESHOOTING
================================================================================

ISSUE: "python is not recognized as an internal or external command"
FIX: Python is not in PATH. Reinstall Python and check "Add to PATH"

ISSUE: "pip is not recognized"
FIX: Use "python -m pip" instead of "pip"
     Example: python -m pip install -r requirements.txt

ISSUE: PowerShell execution policy error
FIX: Run: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

ISSUE: "Address already in use" error on port 8000
FIX: Use a different port:
     python main.py --api --port 8080
     Then go to http://localhost:8080

ISSUE: API quota exceeded error
FIX: Wait a few minutes and try again (free tier has rate limits)

ISSUE: Module not found error
FIX: Make sure virtual environment is activated (you see (venv) in prompt)
     Then reinstall: pip install -r requirements.txt

================================================================================
                          API ENDPOINTS
================================================================================

When running in API mode, these endpoints are available:

GET  http://localhost:8000/          - Web UI
GET  http://localhost:8000/api       - API information
GET  http://localhost:8000/health    - Health check
GET  http://localhost:8000/tools     - List available tools
POST http://localhost:8000/task      - Submit a task

Example API call with curl:

    curl -X POST http://localhost:8000/task -H "Content-Type: application/json" -d "{\"task\": \"Weather in Tokyo\"}"

================================================================================
                       PROJECT ARCHITECTURE
================================================================================

The application uses a 3-agent architecture:

1. PLANNER AGENT (ü§ñ)
   - Analyzes your natural language request
   - Creates a step-by-step execution plan
   - Selects appropriate tools for each step

2. EXECUTOR AGENT (‚ö°)
   - Executes each step in the plan
   - Calls GitHub API and Weather API
   - Handles retries on failures (up to 3 attempts)

3. VERIFIER AGENT (‚úîÔ∏è)
   - Validates execution results
   - Formats the final response
   - Ensures completeness and quality

Available Tools:
- GitHub: Search repos, get repo details, get user info
- Weather: Current weather and 7-day forecast (worldwide)
